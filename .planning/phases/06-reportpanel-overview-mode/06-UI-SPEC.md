## Card Layout

Single-column stacked layout. 520px panel width; cards stack vertically with 16px gap.

**ChapterCard anatomy (top to bottom):**

```
┌─────────────────────────────────────┐  ← border: 1px solid --c-edge
│  [Icon 20px]  Label (1.05rem 600)   │  ← 8px top pad, 16px left/right pad
│                                     │
│  [Display metric]  [unit label]     │  ← JetBrains Mono 1.5rem + Karla 0.85rem muted
│                                     │
│  [StatusDot]  [severity label]      │  ← 8px bottom pad
└─────────────────────────────────────┘
  min-height: 44px (accessible touch target enforced via padding)
  border-radius: 10px (--r-md from tailwind config)
  background: --c-panel
  hover: box-shadow 0 0 0 2px rgb(--c-cyan / 0.25)
  cursor: pointer
```

Cards are NOT in a CSS grid. `display: flex; flex-direction: column; gap: 16px` on the ReportPanel scroll container.

## Interaction Contract

### Card Click (Drill-Down)
- Trigger: click anywhere on ChapterCard
- Effect: calls `onDrillDown(tabId)` → App.tsx runs `setViewMode('advanced')` + `handleSelect(tab)` atomically
- No confirmation. No hover tooltip needed.
- Source: CONTEXT.md D-06

### Back to Overview
- Only affordance: OVW/ADV toggle button in ModuleRail (Phase 5)
- No breadcrumb, no back link on ReportPanel itself
- Source: CONTEXT.md D-07

### View Transition (Overview ↔ Advanced)
- Mechanism: `AnimatePresence mode="wait"` nested inside ModuleSheet's `motion.aside`
- Animation: `initial={{ opacity: 0 }} animate={{ opacity: 1 }} exit={{ opacity: 0 }}` — duration 150ms ease
- Source: RESEARCH.md Pattern 5

### Panel Header in Overview Mode
- Replace module-specific header with plain "Location Report" heading (Karla 1.2rem semibold)
- Module-specific header only renders when `view === 'advanced'`
- Source: RESEARCH.md Pitfall 5

## States

### No Location Set (Placeholder)
- 6 cards render with icon + label visible
- Metric area shows `--` in `--c-muted` color
- Below the 6 cards: muted body text — "Set a location to see data"
- StatusDot: not shown in placeholder state (no dot at all)
- Source: CONTEXT.md D-08

### Loading (Location set, data fetching)
- Metric area: `LoadingSkeleton` component in place of metric value
- StatusDot: not shown until `status === 'success'`
- Other cards that resolve earlier render immediately
- Source: RESEARCH.md Pitfall 4

### Loaded (Success)
- Metric value shown in JetBrains Mono Display size
- StatusDot renders with correct tone + label
- Card is interactive (pointer cursor, hover ring active)

### Partial API Failure
- Failed card: icon + label visible; metric area shows "Data unavailable" in `--c-muted` at Body size; card background opacity 60% (`opacity-60`)
- Failed card: NOT interactive (cursor default, no hover ring)
- Other cards render normally and remain interactive
- Source: CONTEXT.md D-09
