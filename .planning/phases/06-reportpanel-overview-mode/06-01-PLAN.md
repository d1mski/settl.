2. **Type export:**
```typescript
export type OverviewSeverity = 'ok' | 'watch' | 'alert' | 'unavailable';
```

3. **Six derive functions**, each accepting `ModuleState<T>` and returning `{ severity: OverviewSeverity; metric: string | null; unit?: string }`:

- `deriveClimateSeverity(state: ModuleState<ClimateData>)`:
  - If `status !== 'success'` or `data === null`: return `{ severity: 'unavailable', metric: null }`
  - Use `countExtremeDays(data.daily)` to get heat days count (import from climateAggregation)
  - Metric: max temperature from `Math.max(...data.daily.temperature_2m_max)` formatted as `"${val.toFixed(0)}"`
  - Unit: `"°C"`
  - Thresholds per D-05: heat days > 30 = alert, heat days 5-30 = watch, else = ok

- `deriveWindSeverity(state: ModuleState<ClimateData>)`:
  - If `status !== 'success'` or `data === null`: return `{ severity: 'unavailable', metric: null }`
  - Metric: `Math.max(...data.daily.wind_gusts_10m_max)` formatted as `"${val.toFixed(0)}"`
  - Unit: `"km/h"`
  - Thresholds: > 90 = alert, 60-90 = watch, < 60 = ok

- `deriveSunSeverity(state: ModuleState<ClimateData>)`:
  - If `status !== 'success'` or `data === null`: return `{ severity: 'unavailable', metric: null }`
  - Metric: `Math.max(...data.daily.uv_index_max)` formatted as `"UV ${val.toFixed(0)}"`
  - Unit: `""` (empty — "UV" prefix baked into metric string)
  - Thresholds: UV >= 8 = alert, UV 6-7 = watch, UV <= 5 = ok

- `deriveHazardsSeverity(earthquakes: ModuleState<EarthquakeEvent[]>, wildfires: ModuleState<WildfireEvent[]>)`:
  - If both `status === 'error'`: return `{ severity: 'unavailable', metric: null }`
  - If either still loading and the other has no data: return `{ severity: 'unavailable', metric: null }`
  - Call `synthesiseRisks({ coords: null, climate: null, features: null, wiki: null, earthquakes: earthquakes.data, wildfires: wildfires.data, aqi: null, elevation: null })` to get Risk[]
  - If no risks: metric `"LOW"`, severity `ok`
  - If any risk has severity `'critical'` or `'warn'`: metric `"HIGH"`, severity `alert`
  - If any risk has severity `'watch'` or `'info'`: metric `"MOD"`, severity `watch`
  - Unit: `""` (empty)

- `deriveAirSeverity(state: ModuleState<AqiSample[]>)`:
  - If `status !== 'success'` or `data === null` or `data.length === 0`: return `{ severity: 'unavailable', metric: null }`
  - Mean PM2.5: `data.reduce((s, d) => s + d.pm2_5, 0) / data.length`
  - Metric: `"${mean.toFixed(1)}"`
  - Unit: `"ug/m3"` (plain text micro-gram approximation — avoid unicode issues)
  - Thresholds per WHO: > 15 = alert, 5-15 = watch, <= 5 = ok

- `deriveContextSeverity(state: ModuleState<NearbyFeature[]>)`:
  - If `status !== 'success'` or `data === null`: return `{ severity: 'unavailable', metric: null }`
  - Count: `data.length`
  - Metric: `"${count}"`
  - Unit: `"places"`
  - Severity: always `ok` (context is informational, rarely alerts — per RESEARCH.md)

All functions must be pure (no hooks, no side effects). Import types from `../types` and functions from `./riskSynthesis` and `./climateAggregation`.

**Props interface:**
```typescript
import type { LucideIcon } from 'lucide-react';

export interface ChapterCardProps {
  icon: LucideIcon;
  label: string;
  metric: string | null;       // null = loading; '--' = no location
  unit?: string;
  severity: 'ok' | 'watch' | 'alert' | 'unavailable';
  onClick: () => void;
}
```

**Rendering logic — 4 states:**

1. **No location (metric === '--'):**
   - Icon (20px, `text-muted`) + Label (Karla 1.05rem/600 semibold, `text-ink`)
   - Metric area: `"--"` in `text-muted` at Display size (JetBrains Mono 1.5rem)
   - No StatusDot rendered
   - Card NOT interactive: `cursor-default`, no hover ring, no onClick firing

2. **Loading (metric === null):**
   - Icon + Label as above
   - Metric area: render `<LoadingSkeleton />` component (import from `./LoadingSkeleton`)
   - No StatusDot rendered
   - Card NOT interactive during loading

3. **Success (metric is a string value, severity is ok/watch/alert):**
   - Icon (20px, `text-ink`) + Label
   - Metric: value in JetBrains Mono 1.5rem `text-ink`, unit in Karla 0.85rem `text-muted` with 4px gap
   - StatusDot: `<StatusDot tone={SEVERITY_TONE[severity]} pulse={severity === 'alert'} label={severity === 'ok' ? 'OK' : severity === 'watch' ? 'WATCH' : 'ALERT'} />`
   - Import `SEVERITY_TONE` from `../../utils/overviewSeverity`
   - Card IS interactive: `cursor-pointer`, hover ring `box-shadow: 0 0 0 2px rgb(var(--c-cyan) / 0.25)`, onClick fires

4. **Unavailable (severity === 'unavailable'):**
   - Icon + Label visible
   - Metric area: `"Data unavailable"` in `text-muted` at Body size (Karla 0.85rem) — per D-09
   - Card at `opacity-60`
   - Card NOT interactive: `cursor-default`, no hover ring, no onClick firing

**Card container classes (Tailwind):**
```
bg-panel border border-edge rounded-[10px]
px-4 py-2
min-h-[44px]
transition-shadow duration-150
```
Add hover ring only when interactive: `hover:shadow-[0_0_0_2px_rgb(var(--c-cyan)/0.25)]`

**Layout inside card:** Three rows using flex column with 4px gap between rows:
- Row 1: `flex items-center gap-1` — icon (w-5 h-5) + label span
- Row 2: `flex items-baseline gap-1` — metric value + unit span
- Row 3: StatusDot (only when severity is ok/watch/alert)

Do NOT add `z-index`, `position: relative`, or `isolation: isolate` to the card — per STATE.md z-index constraint.
